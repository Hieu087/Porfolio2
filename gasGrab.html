Gas Grab

Details of my works

Gas Grab is a gas order service app. Making the tracking order map is one of my main tasks in this project, but thanks to the help of Google API, everything was executed smoothly.

View in store

1. Login Function

- Create login function with OTP confirmation by using provider from Google Firebase.
- Create an OTP confirmation screen in order to ensure users are providing a valid phone number.

2. Order Function

Create gas order function, this will come with 4 screens:
- Confirm your cart: Users can have multiple items in their cart, the price will be calculated base on each item and its quantity.
- Confirm your address and time of delivery: If users didn't enter their address before, the system would force them to enter it in order to continue.
- Final confirmation and note for delivery associates: Customers's note will display on delivery associates's application.
- Order complete: Just a screen says thank you for your support.

3. Addresses Picker

There are 2 options for users:
- Choose address by entering and picking a suitable address from the addresses list, the system will suggest a list of addresses that suit the words users enter. This has been done with Google autocomplete API.
- Allow the app to get their location and use their current location as the address. This has been done with react-native-geocoding module.

4. Order Tracking

- Users are able to track their order.
- Users will get a notification as soon as the delivery starts. By clicking on the notification, users will be navigated to a screen which displays a map with their address's location, delivery associates's location and info, there also a colored path between their locations.
- Delivery associates's location will be updated continuously to the server through API from their app, then users's app will also receive that location from the server through API continuously.
- Google map and map direction API were used to accomplish.
* This function has been broken since I left that why the map isn't showing.

5. Delivery associates's App

- This app are used by delivery associates to receive and deliver orders from customers.
- The distributor will decide which order will be delivered by whom.
- Delivery associates get a notification as soon as there's an order need them to deliver.
- There are 5 stages in delivery procedure: Initial, ready-to deliver, delivering, complete, cancel. Delivery associates have to write a reason in order to cancel the order.
- Delivery associates's app gets users's location by react-native-geocoding module and posts it to the server every 15 second. In the meantime, customers's app gets this location and displays it on tracking map.
